# 上海买房地图 — 设计文档

日期：2026-07-17
类型：个人自用工具

## 目标

一个交互式的上海市区地图，帮助个人研究买房。地图上能看到：

- 全部行政区的位置与划分（大区结构）
- 全市齐全的房产板块（可悬停高亮、点击查看详情）
- 明显区分内环 / 中环 / 外环三条环线
- 重点研究范围：中环以内的板块（含真实房产数据）；外环外板块显示轮廓、可无详细数据

点击某个板块，展示该板块的总结（最近成交均价、优点、缺点）以及可出售小区列表（小区名、均价、最近成交时间等）。

## 整体架构

项目由三部分组成：

1. **地图应用（前端）** — 日常查看的界面。`index.html` + JS + CSS，用 Leaflet 渲染。读取本地地理边界（GeoJSON）、房产数据（`housing.json`）和收藏（`favorites.json`）。
2. **本地轻服务（`server.py`）** — 用 Python 标准库实现的小服务，做两件事：把前端页面/数据发给浏览器；接收前端的收藏请求，写入 `data/favorites.json`。仅在本机运行、不对外，无额外安全风险。通过双击 `start.command` 启动。
3. **抓取脚本（`scrape.py`）** — 需要更新数据时手动运行。抓取范围默认包含收藏内容（读 `favorites.json`）+ 脚本顶部手动配置的 `FOCUS_*`，产出 `housing.json`。与前端解耦——脚本失败时地图仍用旧数据正常显示。

关键原则：**地图与数据解耦**。地理边界做好基本不动；变的是 `housing.json`（抓取产出）与 `favorites.json`（收藏动作写入）。抓取只拿到部分数据时，地图仍能正常展示已有部分。

## 数据结构

### 地理文件（一次性制作，基本不动）

- `districts.geojson`：行政区多边形。属性含区名。
- `blocks.geojson`：板块多边形（用街道边界近似真实房产板块）。属性含：板块名、所属行政区、所属环线（内环 / 中环 / 外环）。
- `rings.geojson`：内/中/外环三条环线，纯参考图层。

### 房产数据（抓取脚本产出，可反复更新）

`data/housing.json`：

```json
{
  "updated_at": "2026-07-17",
  "blocks": {
    "联洋": {
      "avg_price": 78000,
      "pros": ["学区好", "商业成熟"],
      "cons": ["房龄偏老", "单价高"],
      "has_favorite": true,
      "communities": [
        {"name": "仁恒森兰雅苑", "avg_price": 82000, "last_deal": "2026-06", "favorite": true}
      ]
    }
  }
}
```

- 顶层 `updated_at`：数据更新日期，界面角落展示。
- `blocks` 以**板块名为 key**，与 `blocks.geojson` 的板块名对应关联。
- 单价单位：元/㎡。
- 小区的 `favorite: true` 标记该小区是关注小区（来自抓取配置 `FOCUS_COMMUNITIES`）；界面据此置顶 + 星标高亮。
- 板块的 `has_favorite: true` 表示该板块下含关注小区；界面据此在地图上给板块加标记，即便板块本身无其他数据。
- 没有房产数据的板块（如外环外）照常显示轮廓，点开显示"暂无数据"。

### 收藏数据（前端点星写入，抓取脚本读取）

`data/favorites.json`：

```json
{
  "blocks": ["联洋", "碧云"],
  "communities": ["仁恒森兰雅苑"]
}
```

- 前端点星收藏/取消时，通过 `server.py` 写入此文件（增删对应条目）。
- 抓取脚本读取此文件，默认将其中的板块/小区纳入抓取范围。
- 前端加载时读取此文件，用于“只看收藏”视图与星标状态回显（`housing.json` 中的 `favorite`/`has_favorite` 也据收藏在抓取时标注）。

## 交互与界面

三层叠加（从下到上）：

1. **环线**：内/中/外环虚线圈，参考层。
2. **行政区**：粗边界 + 淡底色 + 大区名，一眼可辨大区范围（浦东/闵行/徐汇…）。
3. **板块**：可交互色块，按环线分三档色阶配色；中环内重点板块颜色更实。

交互：

- **悬停板块**：整块高亮 + 轻微放大 + 小提示（板块名 + 均价）。
- **点击板块**：右侧滑出面板，显示板块总结（均价、优点、缺点）+ 可出售小区列表（小区名、均价、最近成交）。
- **关注小区标记**：`favorite` 小区在面板列表中置顶 + 星标高亮；含关注小区的板块（`has_favorite`）在地图上带标记，即便板块本身无其他数据。
- **收藏（点星）**：板块面板和小区列表旁有星标☆，点击即收藏（★），再点取消。收藏动作经 `server.py` 写入 `favorites.json`，关机重开仍在。
- **只看收藏**：界面有一个开关，打开后地图/列表只重点显示已收藏的板块和小区，方便集中查看关注对象。
- **角落信息**：图例 + “数据更新于 YYYY-MM-DD”。

视觉实现将使用 frontend-design 方法，避免套模板的通用观感。

## 抓取脚本

### 抓取范围配置（可调整）

范围配置写在脚本顶部，支持两种模式，可单用或叠加（取并集）：

```python
# ===== 抓取范围配置 =====

# 模式一：按环线圈（前期不了解具体板块时用）
FOCUS_RINGS = ["内环", "中环"]   # 抓这些环线内的所有板块；设为 [] 则不启用

# 模式二：按具体板块（后期锁定重点时用）
FOCUS_BLOCKS = ["联洋", "古北"]  # 精确指定；设为 [] 则不启用

# 模式三：按具体小区（对某个楼盘单独感兴趣时用）
FOCUS_COMMUNITIES = ["仁恒森兰雅苑"]  # 精确抓这些小区，哪怕所在板块没被选中；设为 [] 则不启用
```

用法：

- **前期广撒网**：`FOCUS_RINGS = ["内环", "中环"]`、其余 `[]` → 抓中环以内所有板块。
- **后期精抓**：`FOCUS_RINGS = []`、`FOCUS_BLOCKS = [...]` → 只抓锁定的几个板块，更快更省。
- **盯具体楼盘**：在 `FOCUS_COMMUNITIES` 列出小区名 → 单独抓这些小区，归入其所属板块，并在界面标为关注。
- **混合**：三者可任意叠加，取并集。

**收藏内容默认纳入**：抓取脚本还会读取 `data/favorites.json`，把其中收藏的板块/小区**默认加入本次抓取范围**。最终抓取范围 = 收藏内容 ∪ `FOCUS_RINGS` 命中板块 ∪ `FOCUS_BLOCKS` ∪ `FOCUS_COMMUNITIES`。这样界面上点星收藏的对象，无需再改脚本配置就会被自动刷新。

按环线筛选依赖 `blocks.geojson` 中每个板块的“所属环线”属性：脚本先按环线属性算出板块清单，合并 `FOCUS_BLOCKS`，叠加 `FOCUS_COMMUNITIES` 与 `favorites.json` 的收藏项，去重后抓取。收藏及 `FOCUS_COMMUNITIES` 命中的小区在产出中标 `favorite: true`，其所属板块标 `has_favorite: true`。改配置或改收藏后重新运行脚本即可，不碰其他代码。

### 抓取行为

- Python 脚本，仅抓取上述范围配置命中的板块。
- 温和策略：请求间加随机延时、使用正常 UA、低频运行。
- **原子更新保护**：先写 `housing.json.tmp`，全部字段抓取成功后再重命名覆盖 `housing.json`；中途失败则丢弃 tmp，上次的好数据不受影响。
- 字面加密价格（链家/贝壳的字体加密）在实现抓取时具体攻克；先保证流程与数据结构跑通，必要字段必要时可人工校对。

### 预期管理（诚实说明）

抓取能否稳定拿到全部字段，取决于目标站点当时的反爬强度，无法 100% 保证。但因地图与数据解耦，即使抓取只拿到部分数据，地图仍可正常展示已有部分。抓取属于个人小范围、低频使用。

## 数据更新流程

看地图和更新数据是**两条相互独立的流程**，没有强制先后：看地图不要求先抓数据（读已有 `housing.json`）；更新数据不要求先开服务（`scrape.py` 独立运行）。唯一关联是：更新后需刷新浏览器才显示新数据。

### 看地图（日常）

双击 `start.command` → 自动起服务并打开浏览器到 `http://localhost:8000`。读取已有数据，无需先抓取。

### 更新数据（偶尔，如每周一次）

双击 `update-data.command` → 运行抓取脚本（默认含收藏内容 + 手动 `FOCUS_*` 配置），**跑完停住显示摘要**（本次更新板块数、失败项、更新时间），确认后手动关闭窗口。因反爬可能导致个别板块抓取失败，摘要便于察觉。抓取完成后，刷新已打开的地图页面即可显示新数据。

也可命令行运行：`cd scraper && python3 scrape.py`。

### 调整关注范围

关注范围有两个来源，二者取并集：

1. **界面收藏（推荐日常用）**：在地图上点星收藏/取消板块或小区，写入 `favorites.json`，下次抓取自动纳入。无需改代码。
2. **脚本配置（批量/环线用）**：编辑 `scraper/scrape.py` 顶部的 `FOCUS_RINGS` / `FOCUS_BLOCKS` / `FOCUS_COMMUNITIES`，适合"抓整个内环"这类批量范围。

改动后重新运行抓取（双击 `update-data.command`）、刷新浏览器即可。若想关注的板块地图上根本没有轮廓，需在 `blocks.geojson` 补边界多边形（设计上板块全市齐全，正常无需此操作；确有缺失请告知板块名）。

## 技术选型

- 地图：Leaflet + GeoJSON（可关闭底图，只画自定义图层）
- 应用形态：本地轻服务（`server.py`，Python 标准库）+ 前端页面；双击 `start.command` 启动
- 抓取：Python
- 地理数据来源：行政区/街道边界的公开 GeoJSON（如 DataV 等），实现时实际拉取验证

## 范围之外（YAGNI）

- 不做线上部署、不做云端后端（本地轻服务仅限本机）
- 不做实时抓取
- 不做用户账号、跨设备同步（收藏存本地文件）
- 板块下钻切换视图（方式 2）作为未来可选增强，本期不做
